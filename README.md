# Sessão 10 — Gestão de Sessões para Clínica Estética

**Sessão 10** é uma plataforma de gestão interna voltada para profissionais autônomos, especificamente para a **Clínica Estética Avançada Daniandrade**. O sistema visa centralizar o cadastro de clientes, registro de atendimentos, controle de agenda e gestão financeira, substituindo processos manuais por uma solução digital integrada. O nome é uma referência às sessões que os clientes da clínica agendam.

## 📋 Sobre o Projeto

O desenvolvimento desta plataforma é parte de um projeto de extensão acadêmica aplicado ao gerenciamento de projetos. A solução foi desenhada para atender às necessidades da empreendedora **Lia Mara Grubert**, que atua nas áreas de drenagem linfática, massagem pós-operatória e terapias corporais.

### Problemas Resolvidos
- **Descentralização de Dados:** Fim dos registros manuais e físicos que dificultavam a consulta rápida.
- **Gestão de Agenda:** Automatização do controle de sessões e histórico de atendimentos.
- **Controle Financeiro:** Centralização de receitas e despesas para uma gestão mais segura e eficiente.
- **Risco de Perda de Informação:** Migração para um ambiente digital, reduzindo o retrabalho e o risco de extravio de dados.

### Estrutura do Projeto

A aplicação foi reestruturada para seguir padrões de modularidade e separação de preocupações:

- **css/**: Estilos segmentados (base, componentes e módulos).
- **js/core/**: Gestão de estado e persistência (`AppStorage`).
- **js/utils/**: Utilitários de formatação e máscaras.
- **js/modules/**: Lógica de interface e funcionalidades específicas.
- **index.html**: Ponto de entrada da aplicação.

### Tecnologias Utilizadas

- **HTML5** e **CSS3** (Modularizado).
- **JavaScript (ES6+)** com arquitetura baseada em módulos.
- **LocalStorage** para persistência de dados.

## ⚙️ Como Executar

Por ser uma aplicação estática (Vanilla JS), não é necessário instalar dependências.

1. Clone o repositório ou baixe os arquivos.
2. Abra o arquivo `index.html` em qualquer navegador moderno.
3. Para deploy, basta hospedar os arquivos em qualquer servidor de arquivos estáticos (como GitHub Pages).
